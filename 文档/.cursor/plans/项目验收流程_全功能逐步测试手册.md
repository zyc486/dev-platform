# 项目验收流程（全功能逐步测试手册）

更新时间：2026-04-22  
适用版本：当前工作区 `毕业设计`（后端 Spring Boot :8080 + 前端 Vite :5174 / 或单端口 :8080 静态托管）  
目的：你按本文档逐步验收项目所有核心功能；**任意一步不可用**，只要把“步骤编号 + 现象/截图 + traceId”发我，我就能直接定位并修复。

---

## 0. 使用说明（强烈建议先读）

- **验收方式**：按编号从小到大执行；每一步都包含：
  - **操作**：你需要点什么/输入什么/访问什么
  - **期望结果**：正确时应该看到什么
  - **失败现象**：常见错误表现
  - **定位指向**：优先怀疑的接口/前端文件/后端文件
- **报错时必带信息**（越全越快修）：
  - **步骤编号**（例如：`2.4`）
  - **你访问的 URL**
  - **浏览器控制台报错（F12）**
  - **后端返回的 `traceId`**（接口错误弹窗一般会带 traceId；或 Network 里响应 JSON 的 `traceId` 字段）
- **演示建议**（预答辩最稳）：
  - 推荐用 **单端口**：`http://localhost:8080/` 打开系统（老师更容易理解）
  - 也可用 **开发端口**：`http://127.0.0.1:5174/`（调试方便）
  - 注意：`localhost` 和 `127.0.0.1` 在 OAuth 回调上**不等价**，步骤 `2.5` 会说明

## 1. 环境与启动验收（必须先通过）

### 1.1 启动后端（8080）
- **操作**：在 `dev-platform-backend` 目录执行：
  - `.\mvnw.cmd -DskipTests spring-boot:run`
- **期望结果**：
  - 控制台显示 `Tomcat started on port(s): 8080`
  - 打开 `http://127.0.0.1:8080/api/home/summary` 返回 JSON 且 `code=200`
- **失败现象**：
  - 8080 端口占用、数据库连接失败、启动报错
- **定位指向**：
  - 后端配置：`dev-platform-backend/src/main/resources/application.yml`
  - 数据库初始化：`dev-platform-backend/src/main/java/.../config/DatabaseSchemaInitializer.java`

### 1.2（可选）启动前端开发服务器（5174）
- **操作**：在 `front/front-spa` 目录执行：
  - `npm run dev`
  - 浏览器打开 `http://127.0.0.1:5174/`
- **期望结果**：页面正常加载，接口能访问（走 Vite proxy 到 8080）
- **失败现象**：白屏、跨域、接口 404
- **定位指向**：
  - 代理配置：`front/front-spa/vite.config.ts`

### 1.3 单端口静态托管（8080 直接访问前端）
- **操作**：浏览器打开 `http://localhost:8080/`
- **期望结果**：页面正常加载（能看到顶栏、菜单、首页）
- **失败现象**：白屏、静态资源 404
- **定位指向**：
  - 前端 build 输出路径：`front/front-spa/vite.config.ts`（`outDir` 必须指向后端 `static/`）
  - 后端 SPA 回退：`dev-platform-backend/src/main/java/.../config/SpaHistoryFallbackFilter.java`

## 2. 账号体系与登录态验收

### 2.1 普通注册（邮箱）
- **操作**：进入注册页，注册一个测试账号：
  - 用户名：`qa_user_时间戳`（避免重复）
  - 密码：建议 `Pass@123456`
  - 邮箱：任意（例如 `qa_xxx@test.local`）
- **期望结果**：提示“注册成功”，可跳转登录
- **失败现象**：
  - 提示必须填手机号/邮箱
  - 用户名已存在
- **定位指向**：
  - 后端：`UserService.register`（`dev-platform-backend/src/main/java/.../service/UserService.java`）
  - 前端：`front/front-spa/src/ui/views/RegisterPage.vue`

### 2.2 密码登录
- **操作**：用刚注册账号登录
- **期望结果**：
  - 登录成功后顶部显示用户名
  - 本地保存 token（刷新页面仍保持登录态）
- **失败现象**：登录成功但刷新丢登录态 / 401 循环跳转
- **定位指向**：
  - 前端登录态：`front/front-spa/src/ui/auth.ts`
  - 请求拦截器：`front/front-spa/src/ui/api.ts`
  - 后端鉴权：`dev-platform-backend/src/main/java/.../interceptor/JwtInterceptor.java`

### 2.3 退出登录
- **操作**：点击“退出登录”
- **期望结果**：回到未登录状态，角标清零
- **定位指向**：
  - `front/front-spa/src/ui/auth.ts`
  - `front/front-spa/src/ui/App.vue`

### 2.4 权限边界（未登录访问受限页）
- **操作**：退出后访问：
  - `/me`、`/projects`、`/notifications`、`/dm`
- **期望结果**：自动跳转登录页（或提示需登录）
- **定位指向**：
  - 路由守卫：`front/front-spa/src/ui/router.ts`

### 2.5 GitHub OAuth 登录/绑定（关键答辩点）
> 注意：不要手动打开 `/auth/callback`。必须通过页面按钮发起授权，否则会出现“授权码缺失”。

- **操作 A（未登录 OAuth 登录）**：
  1) 退出登录  
  2) 登录页点“GitHub 登录”  
  3) GitHub 授权后回调到：
     - `http://localhost:8080/auth/callback?code=...&state=...`（单端口演示）
     - 或 `http://127.0.0.1:5174/auth/callback?...`（开发模式）
- **期望结果**：
  - 成功拿到 token 并进入首页/重定向页
- **失败现象**：
  - 回调页提示“授权码缺失”（说明你手动打开了或 GitHub 没带 code）
  - state 校验失败
- **定位指向**：
  - 前端：`front/front-spa/src/ui/views/AuthCallbackPage.vue`、`LoginPage.vue`、`oauthGithub.ts`
  - 后端：`dev-platform-backend/src/main/java/.../controller/UserController.java`（`/oauth/login`、兜底 `/oauth/callback`）
  - GitHub OAuth App 配置：Homepage/Callback URL 必须与演示域名一致（`localhost` vs `127.0.0.1`）

- **操作**：

  - 点击“清空”→ 列表清空且角标变 0

- **定位指向**：

- **定位指向**：
  - 前端 WS：`front/front-spa/src/ui/wsClient.ts`

## 4. 信用画像（核心主线）

### 4.1 公开信用排行
- **操作**：打开“信用/画像”页面，查看排行；或直接请求：
  - `GET /api/credit/rank2?scene=综合&limit=20`
- **期望结果**：返回列表，展示用户名/等级/分数等
- **定位指向**：`CreditScoreController.rank2`

### 4.2 信用查询（V2）
- **操作**：查询任意 GitHub 用户（例如 `torvalds`）
- **期望结果**：返回总分、维度分、等级等；写入查询历史
- **失败现象**：GitHub 不可达/超时/返回空
- **定位指向**：
  - `CreditScoreController.query`
  - `CreditScoreService`、`CreditScoreV2Service`（网络/算法）

### 4.3 AI 画像
- **操作**：打开 AI 画像（公开查询）：
  - `GET /api/credit/aiProfile?githubUsername=torvalds&scene=综合`
- **期望结果**：返回 summary/techTags 等
- **定位指向**：
  - `AiProfileController.aiProfile`
  - `AiProfileService`、`AiApiClient`（AI API 配置/限流）

### 4.4 查询历史（登录后）
- **操作**：进入“个人中心/信用历史”或调用：
  - `GET /api/credit/queryHistory`
- **期望结果**：能看到历史记录；删除单条/批量删除可用
- **定位指向**：`CreditScoreController.queryHistory`、`QueryLogMapper`

### 4.5 导出（CSV/HTML）
- **操作**：
  - `GET /api/credit/export?usernames=xxx,yyy&scene=综合`
  - `GET /api/credit/exportDetailed?usernames=xxx&scene=综合`
- **期望结果**：下载文件成功，中文文件名不乱码，内容可打开
- **定位指向**：`CreditScoreController.export*`
